## Негатив (упор)

| Элемент | Проблема | Как воспроизвести |
|---------|----------|-------------------|
| curl import с shell | При передаче JSON через `-d "{\"content\":\"...\"}"` с переносами строк в content возможна ошибка `invalid_json` (если shell неправильно экранирует) | Использовать python/script для сборки JSON |
| smoke-test credentials | `npm run test:smoke` использует admin@local.dev — 401 | В seed создан admin@rootwork.ru, а не admin@local.dev |

| **Интерфейс (все кнопки)** | Браузерный MCP недоступен, тесты проводились через API | 1. Открыть http://localhost:3000 (или публичный URL) 2. Войти admin@rootwork.ru 3. Пройти инвентарь кнопок из плана |

## Инструкции для ручной проверки (уками)

### Тест виртуального импорта (без API ключа)

```
Шаги: 1) .env: LLM_STUB_MODE=1  2) Перезапустить docker/api  3) Импорт → Текст
Ввод: Каждая строка = задача (≥4 символа), например:
  Задача 001: Настройка окружения
  Задача 002: Миграции БД
  ...
Ожидание: Задачи созданы с stage (A,R1,R2,F), priority 1-4, size
Что проверить: Нет ошибки LLM, fallback не используется (virtual parser)
```

### Тест всех кнопок хедера

```
Шаги: 1) Войти 2) Нажать каждую: Logo, Таймер, Тема, + Задача, Проекты, Импорт, Профиль
Ожидание: Открытие модалок/меню, без JS-ошибок
Что проверить: Консоль браузера (F12)
```

### Тест экспериментов (пустой ввод, спецсимволы)

```
Шаги: 1) Создать задачу с пустым title  2) Ввести в budget: abc  3) Поиск: "
Ожидание: Валидация или понятная ошибка, не краш
```
