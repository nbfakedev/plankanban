# Базы знаний / инструменты для Codex и агентов

Цель: уменьшить галлюцинации и ускорить навигацию по документации/паттернам.

## 1) Context7 (MCP сервер документации)
Что даёт: подтягивает актуальные, версионные фрагменты официальной документации прямо в промпт. Источник: репозиторий Context7 MCP. (источник: см. ссылки в README данного инструмента)
Где пригодится:
- Cloudflare Workers, nginx, Postgres, Node.js библиотеки, любые SDK.
Как использовать:
- В средах/клиентах с поддержкой MCP подключить Context7 MCP и в промптах добавлять “use context7”. (источник: см. ссылки в README данного инструмента)

## 2) ComposioHQ/agent-orchestrator
Что даёт: оркестратор параллельных агентов, отдельные worktree/ветки/PR на агента. (источник: см. ссылки в README данного инструмента)
Где пригодится:
- когда появится много задач на бэкенд/фронт/инфру одновременно и нужно параллелить без ручного “ветка-PR-ревью”.
Сейчас:
- можно отложить до момента, когда будут стабильные CI и тесты.

## 3) czlonkowski/n8n-mcp
Что даёт: MCP сервер с доступом к документации и операциям n8n (тысячи нод). (источник: см. ссылки в README данного инструмента)
Где пригодится:
- если автоматизацию (импорт/экспорт/уведомления/интеграции) будете собирать в n8n.
Сейчас:
- не нужно для базовой серверной версии канбана.

## 4) nextlevelbuilder/ui-ux-pro-max-skill
Что даёт: “skill” для дизайн-интеллекта (паттерны UI/UX) для Claude Code/клиентов. (источник: см. ссылки в README данного инструмента)
Где пригодится:
- при переработке интерфейса (многопользовательский режим, настройки, админка).
Сейчас:
- полезно как справочник/гайд по UI, но не блокирует серверные задачи.

## 5) 0xdesign/design-plugin
Что даёт: плагин для Claude Code “design-and-refine” для генерации/итерации UI. (источник: см. ссылки в README данного инструмента)
Где пригодится:
- быстрые вариации UI-компонентов, когда перейдёте на компонентную архитектуру (React/Vue) или хотя бы на шаблоны.
Сейчас:
- опционально; полезнее после фикса API и ролей.

## 6) supermemoryai/supermemory
Что даёт: “универсальная память” через MCP сервер (хранение/извлечение контекста между сессиями). (источник: см. ссылки в README данного инструмента)
Где пригодится:
- хранить “долгую память” проекта для агентов: принятые решения, ADR, правила код-стайла, регрессии.
Сейчас:
- полезно сразу, если Codex/агенты поддерживают MCP-клиент; иначе — можно отложить.

## Рекомендованный порядок подключения
1) Context7 (для документации по стеку).
2) Supermemory (для долгой памяти по проекту).
3) UI/UX skills/plugins — после стабилизации API.
4) n8n-mcp — когда решите использовать n8n.
5) agent-orchestrator — когда появится CI и понадобится параллелить задачи.
